# nashville-transit-performance
Python and SQL analysis of WeGo transit data, focusing on on-time performance and logistical  efficiency
### What is this project?
I’m currently a student in the NewForce Data Analytics bootcamp, and for this project, I really wanted to dive deep into transportation data. Since I used to be a FedEx Express DOT Courier at the MTNA station, I know firsthand how frustrating it is when a route gets behind schedule. I used this project to see if the data could help explain why those delays happen.
### What I did:
I took a massive dataset of "Headway Data" from the Nashville transit system and used my new technical skills to find some real answers:
Python (Pandas): I used this to clean up the messy data and group everything together so it actually made sense.
Seaborn & Matplotlib: I made charts to visualize exactly where the buses were getting stuck.
SQL: I ran queries to filter out the noise and focus on the most important transit metrics.
### Takeaways:
Overall, the system is on time about 77.08% of the time, which shows there's definitely room for improvement.
I found that if a bus starts even 10 minutes late, it’s almost impossible to catch up. In my experience on the road, once you're behind, the clock just stays against you.
Most of the time is actually lost in the middle of the trip, not just at the start or end.
